# Skill: Retrospective

Extracts learnings from a completed task and proposes improvements to the framework.

## What It Does
- Reviews the full task history (context, plan, changelog, feedback, verification)
- Produces a retrospective in learnings/retrospectives/
- Identifies patterns, context gaps, and errors
- Proposes specific improvements to framework modules or knowledge-base
- After human approval, applies improvements to the framework repo
